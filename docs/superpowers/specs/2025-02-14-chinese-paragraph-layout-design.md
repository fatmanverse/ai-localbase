# 中文正文段落排版兜底设计

## 背景
当前聊天回答在部分平台回复内容中会出现中文标点被单独挂到下一行的问题，例如顿号 `、`、逗号 `，`、冒号 `：` 等。该问题在流式输出、Deferred fallback 和最终 Markdown 渲染阶段都可能出现，影响中文阅读体验。

## 目标
仅对**普通正文段落**做轻量排版兜底，改善中文阅读感，避免以下问题：
- 中文标点单独成行
- 行首出现明显应归并到上一行的中文标点
- 正文段落出现少量明显异常的碎裂断行

本次增强采用**偏保守策略**：
- 优先修复明显误断行
- 不主动重排作者有意保留的分段
- 宁可少修，也避免误伤 Markdown 结构和语义

## 非目标
以下结构尽量不做主动改写：
- 标题
- 列表项
- 表格
- 引用块
- 代码块
- 图片占位、链接、对象名结构

## 方案选择
采用方案 A：在现有 `textCleanup.ts` 基础上，扩展“正文段落清洗”能力。

原因：
- 改动范围小，便于回归验证
- 可直接复用现有文本清洗链路
- 更容易控制误伤范围
- 同时兼容流式文本与最终 Markdown 渲染

## 设计

### 1. 清洗入口
新增/扩展正文清洗函数，专门处理普通正文段落中的中文断行问题。

接入位置：
- `frontend/src/components/markdown/textCleanup.ts`
- `frontend/src/components/markdown/MarkdownRenderer.tsx`
- `frontend/src/components/markdown/DeferredMarkdownRenderer.tsx`
- `frontend/src/components/ChatArea.tsx`
- `frontend/src/widget/components/MessageList.tsx`

### 2. 清洗规则
按保守顺序执行：
1. 单独成行的中文标点并回上一行
2. 行首中文标点并回前一行末尾
3. 对明显异常的正文短断行做极保守合并
4. 遇到 Markdown 结构边界时停止处理

其中第 3 条只在以下条件同时满足时触发：
- 当前与上一行都位于同一个普通正文块内
- 上一行末尾不是明显自然收束，例如未以完整句号、问号、感叹号或空行结束
- 当前行长度较短，表现为补语、续写、半句说明，而不是完整新段落
- 当前行开头不是标题、列表、引用、表格、代码围栏等 Markdown 结构形态
- 当前行也不像新的独立段落起句，例如明显的“首先 / 其次 / 总结 / 注意”等起段信号

明确约束：
- 对于“上一行以逗号、顿号、冒号等弱停顿结尾，下一行只是短续写”的情况，可以合并
- 对于“下一行已经是完整长句或明显新段落”的情况，不合并
- 不做跨空行合并

优先覆盖标点：
- `、`
- `，`
- `。`
- `；`
- `：`
- `！`
- `？`
- `）`
- `】`
- `》`
- `」`
- `』`

### 3. 结构边界
清洗逻辑必须避免跨越以下行：
- `#` 标题行
- `-` / `*` / `1.` 等列表行
- `>` 引用行
- 表格相关行
- 代码块围栏

实现上先识别**普通正文块**：
- 连续非空行
- 不属于上述 Markdown 结构
- 不处于代码块围栏内部

只有在同一个普通正文块内，才允许执行标点归并和短续写合并。

### 4. 样式配合
正文容器统一使用更适合中文的断行策略：
- `word-break: normal;`
- `overflow-wrap: break-word;`
- `line-break: auto;`

此调整只作为排版配合，不取代文本清洗。

## 风险与控制

### 风险
- 误把正常换行段落并成一行
- 误碰 Markdown 结构边界
- 流式阶段与最终渲染阶段表现不一致

### 控制措施
- 只在正文段落应用
- 规则以“标点归并”为主，仅补充短续写合并，不做大规模重排
- 保持流式阶段使用轻量清洗，Markdown 阶段再做一次更完整清洗
- 为“短续写合并”单独补回归样例，覆盖应合并与不应合并两侧

## 验证方案
1. 使用存在中文标点悬挂问题的长段落进行回归测试
2. 同时检查：
   - 流式输出阶段
   - Deferred fallback 阶段
   - 最终 Markdown 渲染阶段
3. 核查列表、代码块、表格未被误伤
4. 补充正文块边界样例：
   - 标题后正文不被误并
   - 列表项之间不被误并
   - 表格与引用块不被误并
   - “短续写”被合并，“完整新句”保持分行

## 完成标准
- 普通正文段落中的中文标点不再单独悬挂成行
- 普通正文段落中的短续写异常断行得到保守修复
- 中文阅读体验明显改善
- 非正文结构保持原有语义与排版稳定
- 前端构建通过
