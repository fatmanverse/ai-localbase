# AI LocalBase 系统级审查与本轮改造清单

## 审查范围

本次审查覆盖以下范围：

1. 产品定位与功能边界
2. 知识库导入、解析、索引、检索、回答链路
3. 普通聊天页、工单机器人页、可嵌入前端组件
4. 对外聊天 API、反馈闭环、FAQ / 知识缺口 / 低质量回答机制
5. 图片型知识处理
6. 文档、接入方式、静态可用性

---

## 总体结论

### 合理性判断

- **产品定位整体合理**：当前仓库已从“本地知识库聊天页”扩展为“知识库 + 外接聊天 API + 工单机器人 Widget + 反馈闭环”的复合型产品。
- **主链路基本闭合**：上传文档 → 解析 / OCR / 入库 → 检索问答 → 点赞 / 点踩 → FAQ / 知识缺口 / 低质量回答沉淀，已经具备最小闭环。
- **架构层次基本清晰**：
  - 后端：`handler / service / store / util / model`
  - 前端：普通聊天页 + Service Desk Widget + Demo 页面
- **外部接入能力已具备基础**：后端已有 `/api/service-desk` 统一接口，前端已有可复用 Widget。

### 主要问题结论

- **普通聊天页原先更偏“本地控制台”，不是天然的外接机器人 UI**，因此会话与知识库绑定逻辑需要收敛。
- **前端嵌入方式之前主要是“复制 React 组件源码”**，缺少“通过已部署前端地址直接嵌入”的方式，这会抬高接入门槛。
- **运营分析仍偏摘要级**，缺少更细粒度的分页、筛选、状态流转接口。
- **图片知识处理已达到 MVP，但仍属于 OCR + 规则增强路线**，尚未进入真正多模态向量与联合排序阶段。

## 本轮纳入并完成的改造

### 1. 普通聊天页：会话固定绑定知识库

已完成：

- 新建会话先弹窗选择知识库
- 每个会话固定绑定一个知识库
- 会话标题旁显示当前知识库
- 支持后续手动切换当前会话知识库
- 发送消息时严格使用当前会话绑定的知识库

### 2. 前端地址嵌入：新增 URL 驱动嵌入页

已完成：

- 新增更简洁嵌入入口 `/embed/{knowledgeBaseId}`
- 保留兼容入口 `?embed=1&kb=...`
- 兼容 `?mode=service-desk-embed&knowledgeBaseId=...`
- 支持通过 AI LocalBase 前端访问地址进行接入
- 支持固定知识库参数化
- 支持 `ticketId / userId / tenantId / sourcePlatform / category / priority / quickPrompts` 等参数
- 嵌入页固定知识库，不提供切换能力

### 3. 图片知识检索：补齐图片原子 chunk 元数据与图片问法召回增强

已完成：

- 文档 chunk 新增 `chunkType / chunkProfile / topic` 元数据
- 图片知识段在入索引时明确标记为 `chunk_type=image`
- FAQ / 操作步骤 / 参数 / 排障 chunk 也会写入结构化类型
- 对旧索引 payload 兼容回填 chunk 元数据，避免历史数据直接失效
- 当用户问题明显是在问截图 / 页面 / 按钮 / 流程图时，会额外补一轮图片 chunk 召回
- 重排阶段会对图片型 chunk 做轻量加权，降低“正文压住截图知识”的情况
- 图片问法下会适度放宽单文档命中限制，避免正文 chunk 把图片 chunk 全部挤掉

### 4. 运营治理：补齐状态流转接口与最简工作台

已完成：

- 新增 FAQ / 知识缺口 / 低质量回答状态更新接口
- 支持 `candidate / approved / ignored`、`pending / resolved / ignored`、`open / resolved / ignored` 等状态流转
- 前端新增 `?mode=ops-console` / `/ops` 最简治理页
- 可直接查看 FAQ 候选、知识缺口、低质量回答和差评反馈
- 可直接在页面上完成基础状态流转

## 当前保留为后续待完善项

### 1. FAQ / 知识缺口 / 低质量回答运营后台

当前已有数据沉淀、摘要接口和基础列表接口，但还缺：

- 状态流转
- 指派 / 备注
- 运营侧操作页
- 更细的分页游标与批量处理动作

### 2. 真正多模态检索增强

当前已实现：

- 图片提取
- OCR
- 图片分类
- 图片描述
- 图文关联
- 图片信息参与检索与回答

但后续仍建议补：

- 多模态 embedding
- 图片向量索引
- 图文联合排序
- 更强的图片摘要模型接入

### 3. 真正 sparse / hybrid search

当前已做 dense + fallback 混合式路线，但后续建议升级为：

- 真实 sparse vector 检索
- 更稳定的 RRF / hybrid ranking
- 多源检索质量指标可观测化
